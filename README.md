# ZoPlus - 论文管理软件现代化

基于 Tauri + Rust + React 重构 Zotero 前端，集成 MiniMax AI 与阿里云云同步的跨平台论文管理桌面软件，开源免费，对标 NoteExpress。

**版本**：v2.0（支持完全动态数据库自省系统）

---

## 核心特性

### 1. 现代化技术栈
- **Tauri**：使用系统原生 WebView，内存占用控制在 50MB 以内
- **React + TypeScript**：现代化前端界面，支持亮色/暗色主题
- **Rust 后端**：性能敏感任务使用 Rust 处理，保证内存安全和执行效率

### 2. 完全动态数据库自省系统（v2.0 新增）
- 运行时自动扫描 Zotero 数据库的实际表结构
- 动态构建 SQL 语句，不依赖硬编码表名/字段名
- 支持不同版本的 Zotero 数据库
- 全局元数据缓存机制，避免重复扫描
- 智能表名候选列表（处理 Zotero 版本差异）

### 3. AI 智能功能
- **文献摘要**：调用 MiniMax API 生成文献摘要
- **文献翻译**：标题和摘要翻译
- **自动标签**：基于文献内容自动生成分类标签
- **文献问答**：针对单篇或跨文献的智能问答
- **智能笔记**：自动生成文献笔记
- **文献对比**：多篇文献对比分析

### 4. 全文搜索
- 基于 Rust Tantivy 搜索引擎
- 毫秒级搜索响应
- 支持中文分词
- 搜索结果高亮显示

### 5. PDF 标注
- 支持多种标注类型：高亮、矩形、椭圆、箭头、自由绘制、文本注释
- 标注数据以 JSON 格式存储
- 标注编辑、删除和导出功能

### 6. 云同步
- 基于阿里云服务器的自定义同步服务
- 支持数据库和附件的增量同步
- 同步冲突检测和解决

## 前置环境

### 必需运行环境
1. **Node.js**：≥ 18.0.0（LTS 版本）
2. **Rust**：最新稳定工具链（通过 rustup 安装）
3. **Tauri CLI**：最新稳定版
4. **Zotero**：≥ 6.0（必须已安装并初始化数据库）
5. **Git**：版本控制

### 第三方服务依赖
1. **MiniMax API**：用于 AI 摘要、翻译、自动标签、文献问答、元数据补全
2. **阿里云服务器**：项目云同步服务

### 环境变量配置
在项目根目录创建 `.env` 文件：

```env
# MiniMax API 配置
MINIMAX_API_KEY=your_minimax_api_key_here
MINIMAX_MODEL=abab5.5s

# 阿里云同步服务配置
ALIYUN_SERVER_URL=https://your-server-ip:port
ALIYUN_SYNC_TOKEN=your_sync_token_here
```
